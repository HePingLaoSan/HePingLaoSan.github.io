仿写APP

年前的时候有了仿写锤子科技官方论坛APP的想法，一时兴起，便把大概的功能（除了账号相关的登陆注册）都搭起来了，但是最近仿写的动力不是太足了，需要列计划，梳理一下开发测试以及上线的时间点。也让自己重整旗鼓，坚持下来。其实这个app整体来说不难，但是进行到现在有几处需要在今后的仿写中引以为戒的：

  1. 仿写前要认真观察每个接口抓包数据，很多接口并不是你想象的那样规范。这个太重要了，如果你没有去仔细观察，而是主观臆断的话，就会像我一样，写了一半，重新来过。。。浪费时间。同时也影响工作效率。
  2. 注册和登陆目前没有是因为接口采用HTTPS加密传输了，传输的数据并没有抓到，这一点还需要Google下。本来想另辟蹊径抓web版的数据来曲线救国，但是还是需要时间去观察下数据，不知收效如何。
  3. 多观察以及研究目标APP的实现，不要急于开工，梳理好各个页面的实现后再开始，会事倍功半。
  4. 这次仿写采用了storyboard，确实效率提升不少，在以后的工作以及编程中可以多多尝试。
1.0版本时间安排
  1. 2月20号 - 2月26号 把各个论坛模块功能编写完成。模块包括：首页板块，论坛其他板块，帖子详情页板块。
  2. 2月27日 - 3月5日 完善界面UI, 研究登陆注册的接口数据
  3. 3月6日 - 3月12日 app加入登陆注册模块
  4. 3月13日 - 3月19日 开始小范围测试，为上线做准备。
  5. 3月20日 - 3月26日 修复bug，继续测试。
  6. 3月27日 - 3月31日 提交App Store，准备发布
  7. 4月1日 APP上线

PS：帖子详情页里，官方app点击会有大图显示，这个需求移到下一版本去做，下一版本主要的修复点就是优化帖子详情页的技术实现。

好久没更博了，督促一下自己。

2017年4月1号更新

今天是4月1号，正好想更个博客，看到二月份写的时间安排今天是app上线的日子，真是讽刺，理想总是和现实有差距的。po 一下当前进度吧，帖子详情板块已经用TextKit重构了，详情页显示目前测试来说没有什么太大问题，图片可以点击查看高清大图，同时也做了cache，节省用户流量，支持了下拉加载更多评论。首页目前做的差不多了，切换各个板块功能上也没有问题。接下来需要做的工作：

  1. 加入用户登录注册部分，目前只打算做登录，注册直接url跳转到web(时间未知，需要再进行抓包尝试，如果还是不行，则需要咨询内部人士了，时间预留两周吧)
  2. 优化下流畅性，详情页滑动的时候会有卡顿（这个就是个长期战争了，和上线时间可同步进行）

这样看来，上线的计划估计要推迟2周了，大概在4月中旬，加油！
